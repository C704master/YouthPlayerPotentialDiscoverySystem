## 改进 1：年龄分收窄

### 现状

```
age_15_score: 100
age_21_score: 60
权重: 15%
```

17 岁 vs 21 岁的年龄分差 = 40 分，乘以 15% 权重后对总分影响 = **6 分**。Top 10 中 6 人 ≤19 岁。

### 问题

- 年龄分跨度太大（100→60），变相惩罚 20-21 岁球员
- 一个 21 岁的世界级球员（如 Bellingham、Wirtz）即使数据全面碾压 17 岁球员，也难以用 6 分的年龄劣势追回来
- "年轻即正义"的倾向过强，但对球探来讲 17 岁和 18 岁的差距远小于 20 岁和 21 岁的差距

### 方案 A（保守）：收窄范围

```yaml
age_scoring:
  min_age: 15
  max_age: 21
  age_15_score: 95
  age_21_score: 70      # 从 60 提升到 70，差距从 40 缩小到 25
```

效果：年龄分对总分影响从 6 分降到 **3.75 分**（25 × 15%）。

### 方案 B（激进）：收窄 + 降权重

```yaml
weights:
  age_score: 0.10        # 从 15% 降到 10%

age_scoring:
  age_15_score: 90
  age_21_score: 70       # 差距 20 分
```

效果：年龄分对总分影响降至 **2 分**（20 × 10%）。释放的 5% 权重可分配给核心表现。

### 建议

**先实施方案 A**。Bellingham 当前 60 分年龄分 → 改为 70 分后总分为 72.6，排名从 #48 升至约 #30-35。效果可量化验证。

## 改进 2：年龄精度从"整岁"升级为"岁+天数"

### 现状

年龄用整岁：`age` 字段为 int（17、18、19...），年龄分是离散阶梯函数。17 岁零 1 天和 17 岁 364 天得分完全相同。

### 可行性

Transfermarkt 数据集（`davidcariboo/player-scores`）的 `players.csv` 包含完整 `date_of_birth`（如 `2007-07-13 00:00:00`），匹配率 92%。

```python
# 伪代码
from datetime import date
ref = date(2026, 7, 1)  # 2026 年 7 月参考日
dob = pd.to_datetime(row["date_of_birth"])
age_days = (ref - dob.date()).days
age_exact = age_days / 365.25  # 浮点年龄，如 18.73 岁
```

### 方案

年龄分由离散阶梯改为连续线性：

```python
# 旧：离散
age_score = 100 if age == 15 else 93 if age == 16 else ...

# 新：连续
age_score = age_15_score - (age_exact - 15) / (21 - 15) * (age_15_score - age_21_score)
```

示例：17 岁 300 天（≈ 17.82 岁）得分为 `100 - (17.82-15)/6 × 40 = 81.2`，而非原来的 87。

### 注意事项

- TM `date_of_birth` 需在 `stage0_collect()` 的 `attach_transfermarkt_data()` 中一并提取
- 未匹配到 TM 的 8% 球员回退到 `born_`（年份），在 `ensure_age()` 中按 `date(年份, 7, 1)` 估算

## 改进 3：联赛强度修正

### 现状

联赛分固定为 50（占位值，等阶段 5），五大联赛完全平等。导致 Ligue 1 在 Top 50 中占 36%。

### 问题

法甲整体竞技水平低于英超/西甲，同一条 per90 数据在法甲和英超的含义不同。不加修正会系统性地高估低强度联赛的球员。

### 方案：Opta Power Rankings 映射

参考 Opta 2024-25 赛季俱乐部平均实力排名，映射到联赛系数：

| 联赛 | Opta 均分（估算） | 联赛分（0-100） | vs 当前 50 |
|------|------------------|-----------------|------------|
| Premier League | ~88 | **90** | +40 |
| La Liga | ~85 | **85** | +35 |
| Bundesliga | ~83 | **80** | +30 |
| Serie A | ~82 | **78** | +28 |
| Ligue 1 | ~80 | **75** | +25 |

联赛分权重 10%，则 Ligue 1 球员比英超球员少拿 **(90-75) × 10% = 1.5 分**。影响温和但方向正确。

### 实施

阶段 5（`league_correction.py`）实现：
1. 创建 `config/league_strength.yaml`，存联赛→Opta 分数映射
2. 在 scorer 中替换 `out["league_score"] = 50.0` 为实际查表

> **协商点**：上述联赛系数是估算值。是否需要我查一下 Opta Power Rankings 的准确数据？或者先用估算值跑一版看效果？

## 改进 4：行为风格分去饱和

### 现状

行为分 = 4 个子维度百分位均值 × 权重之和，再 clip(0, 100)。Top 50 中 44 人 = 100 分。

### 问题

百分位均值天然接近 50，但对顶尖球员来说每个维度都高，加权和后轻松超过 100 被 clip 掉。clip 成为常态意味着分数失去了区分度。

### 方案 A：木桶效应（最低值法）—— ❌ 已否决

```python
behavior = min(attack_pct, prog_pct, defense_pct, discipline_pct) * 100
```

否决原因：边锋的天职是进攻，要求一个边锋的防守百分位 = 中场水平不合理。Yamal 进攻 98 分但因防守 15 分 → 行为分 = 15，过于苛刻。

### 方案 B：降 clip + 提权 —— ❌ 已否决

```python
raw = attack_pct * 0.22 + prog_pct * 0.18 + defense_pct * 0.18 + discipline_pct * 0.12
behavior = min(raw, 100)
```

否决原因：本质上仍是加权平均，只是把分数整体压低了。并没有改变"偏科生和全能生看起来差不多"的结构性问题。

### 待设计：方案 C

> 待后续讨论。核心矛盾是：**如何在奖励全能的同时，不惩罚符合位置职责的合理偏科（如边锋防守少、DM 进攻少）。**
>
> 可能方向：分位置的"必修维度" vs"加分维度"——必修维度用木桶，加分维度用加权，两者混合。

## 改进 5：纪律分细化

### 现状

```yaml
risk_penalty:
  rules:
    high_yellow_per90: 1.5      # 一刀切阈值
    high_yellow_penalty: 2      # 固定扣 2 分
    any_red_card_penalty: 3     # 一张红牌扣 3 分
```

### 问题

- 黄牌 per90 = 1.49 不扣分，1.51 扣 2 分——阈值边界太硬
- 红牌全扣 3 分，不管是因为战术犯规还是暴力行为
- 纪律性已反向计入行为风格分（discipline 子维度），与风险扣分有重复计算

### 方案

**风险扣分只保留红牌，黄牌移入行为风格**：

```yaml
risk_penalty:
  rules:
    any_red_card_penalty: 3         # 保留，直红是严重信号
    second_yellow_red_penalty: 2    # 新增：两黄变一红
    # 删除 high_yellow_per90 规则

behavior:
  sub_weights:
    discipline: 0.25                # 从 0.20 提升，补偿删除的黄牌扣分
```

同时在 discipline 子维度中做梯度计算：

```python
# 黄牌 per90 → 纪律分（连续映射，无硬阈值）
yp90 = yellow_cards / minutes * 90
# yp90 = 0 → 纪律分 100
# yp90 = 2.0 → 纪律分 0
discipline_score = max(0, 100 - yp90 * 50)
```

效果：黄牌多不再是突然扣 2 分，而是连续拉低行为风格分，更平滑。

## 改进 6：位置权重再平衡

### 现状

CM/DM 的防守指标（tackles_per90, interceptions_per90）权重已经较高，但核心表现分的计算方式是**同位置内部百分位排名**——一个 DM 的抢断再多也只和 DM 比。

### 问题

- CM/DM 的核心表现分不低，但行为风格分中"进攻"维度拖后腿
- DM 在 attack（goals+shots+dribbles+key_passes）维度天然吃亏

### 方案

行为风格的子维度权重按位置分异：

```yaml
behavior:
  sub_weights_by_position:
    Winger: {attack: 0.35, progression: 0.30, defense: 0.10, discipline: 0.25}
    AM:     {attack: 0.30, progression: 0.30, defense: 0.15, discipline: 0.25}
    CM:     {attack: 0.20, progression: 0.25, defense: 0.30, discipline: 0.25}
    DM:     {attack: 0.10, progression: 0.20, defense: 0.45, discipline: 0.25}
```

### 建议

这是一个明显的改进方向，但会增加 scorer 复杂度。**可以等前 5 项改进落地后看效果再决定是否做**。

## 改进 7：百分位池扩大

### 现状

同位置核心表现仅在**被评分的同位置球员内部**做百分位排名。对于 DM（76 人），76 个人的百分位统计意义有限。

### 方案

短期内不扩大（当前五大联赛 U21 球员池已经够用）。如果未来扩展到更多联赛或用历史数据做跨赛季对比，再引入更大的参考池。
