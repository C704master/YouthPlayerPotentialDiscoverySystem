# 阶段 6 · 候选清单与排行榜

> 对应原文档：五、核心模块设计（main.py 串联）／ 十一、输出文件设计 ／ 十三、开发阶段规划（阶段 6）

## 阶段目标

输出 **100 名候选球员基础样本清单** 和 **Top 20 潜力排行榜**。

## 输入 / 输出

| 项 | 内容 |
|----|------|
| 输入 | 阶段 4/5 的 `scored_players.csv` |
| 输出 | `data/processed/candidate_players_100.csv`、`outputs/rankings/top20_ranking.csv`、`top20_ranking.xlsx` |
| 模块 | `main.py`（串联）+ 排序导出逻辑 |

## 输出物设计

| 输出物 | 格式 | 说明 |
|--------|------|------|
| 100 名候选球员基础样本清单 | CSV / Excel | 包含姓名、年龄、位置、球队、联赛、分钟数、数据完整度 |
| Top 20 潜力排行榜 | CSV / Excel / Markdown | 用于展示最值得关注的年轻球员 |

## 生成规则

1. **候选样本清单**：从符合筛选条件（U21、目标位置、分钟数达标）的球员中，输出至少 100 名基础样本。
   - 字段：姓名、年龄、位置、球队、联赛、分钟数、数据完整度。
   - 若数据不足 100 名，报告中**明确说明**，不得虚构。
2. **Top 20 排行榜**：按 `total_score` **降序**取前 20 名。
   - Top 20 必须满足 `minutes >= 600`（正式评分约束，来自阶段 2）。
   - 同时导出 CSV 与 Excel，便于展示。

## 实现要点

- 使用 pandas `sort_values(by="total_score", ascending=False)` 排序。
- 用 `to_csv` / `to_excel` 导出。
- 候选清单与 Top 20 榜分别落盘，路径遵循目录结构约定。
- 由 `main.py` 负责把阶段 1→6 串联为完整批处理流程。

## 相关风险

| 风险 | 表现 | 解决方案 |
|------|------|----------|
| 样本不足 | 符合条件球员少于 100 名 | 报告中明确说明数据不足，不虚构补足 |

## 验收标准

- [ ] 输出 `candidate_players_100.csv`，包含至少 100 名符合要求球员；数据不足时报告中明确说明。
- [ ] 输出 Top 20 排行榜并按 `total_score` 降序排列。
- [ ] Top 20 中不存在 `minutes < 600` 的球员。

## 学习路径提示

- 学会排序和导出：`sort_values`、`to_csv`、`to_excel`。
