# 阶段 9 · 汇总与答辩材料

> 对应原文档：十一、输出文件设计（汇总报告包）／ 十三、开发阶段规划（阶段 9）／ 十四、学习路径 ／ 十五、验收标准 ／ 十七、后续扩展方向

## 阶段目标

生成**汇总报告包**和 **README**，形成可提交、可答辩的完整交付物。

## 输入 / 输出

| 项 | 内容 |
|----|------|
| 输入 | 前序所有输出（评分、排行榜、雷达图、单人报告） |
| 输出 | `outputs/summary_report.docx`、项目根 `README.md` |
| 模块 | `word_exporter.py`（汇总）+ 文档撰写 |

## 输出物设计

| 输出物 | 格式 | 说明 |
|--------|------|------|
| 汇总报告包 | DOCX | 包含项目说明、评分方法、排行榜和报告索引 |

汇总报告应至少包含：

1. 项目说明（目标、系统边界）。
2. 评分方法说明（评分公式、分项权重、同位置百分位、联赛强度 10% 修正）。
3. Top 20 排行榜。
4. 单人报告索引（指向各球员 md/docx）。

## 项目整体验收清单（第十五章汇总）

| 验收项 | 验收方式 |
|--------|----------|
| 成功读取 CSV | 运行 main.py 后控制台显示读取行数和字段检查结果 |
| 筛选 U21 | 输出数据中不存在 age > 21 的正式评分球员 |
| 筛选位置 | 正式评分数据只包含 Winger、CM、AM、DM |
| 过滤分钟数 | Top 20 中不存在 minutes < 600 的球员 |
| 候选样本清单 | 输出至少 100 名球员；不足时明确说明 |
| 评分范围 | 所有正式评分球员 total_score 在 0-100 之间 |
| 分项解释 | 包含年龄、出场、核心表现、行为风格、联赛强度、风险扣分 |
| 排行榜 | 输出 Top 20 并按 total_score 降序 |
| 雷达图 | Top 20 每名球员至少一张 PNG |
| Word 报告 | Top 20 每名球员单独 DOCX |
| 报告真实性 | 报告不出现未提供的伤病、转会、采访或心理判断 |

## 适合高中生的学习路径（第十四章）

1. 先学会 pandas 读取 CSV：`pd.read_csv`、`df.head`、`df.columns`。
2. 学会筛选数据：`age <= 21`、`minutes >= 600`、`position` 是否在指定列表中。
3. 学会新增列：计算 `goals_per90`、`assists_per90`、`key_passes_per90`。
4. 学会排序和导出：`sort_values`、`to_csv`、`to_excel`。
5. 学会简单函数：把清洗、评分、画图分别写成函数。
6. 学会 matplotlib 画雷达图。
7. 学会 python-docx 生成 Word。
8. 最后再接入大模型生成自然语言报告。

## 后续扩展方向（第十七章，非第一版范围）

- 增加 Jupyter Notebook 教学展示版，便于答辩时逐步解释。
- 增加 Streamlit 本地网页，支持上传 CSV、查看榜单和下载报告。
- 扩展前锋、后卫、门将评分模型。
- 接入更稳定的数据源或 API。
- 增加赛季对比和成长趋势分析。
- 引入更完整的 Opta 或同类数据服务后，自动更新联赛强度。
