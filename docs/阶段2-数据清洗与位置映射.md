# 阶段 2 · 数据清洗与位置映射

> 对应原文档：五、核心模块设计（cleaner.py、position_mapper.py）／ 七、数据清洗规则 ／ 十六、风险与解决方案（小样本虚高）

## 阶段目标

能筛选 U21、位置、分钟数并处理缺失值，输出干净、可评分的数据集，并按位置分组：Winger / CM / AM / DM。

## 输入 / 输出

| 项 | 内容 |
|----|------|
| 输入 | 阶段 1 的原始 DataFrame、`config/config.yaml` |
| 输出 | `data/processed/cleaned_players.csv`、`standard_position` 字段 |
| 模块 | `cleaner.py`、`position_mapper.py` |

## 数据清洗规则（按顺序执行）

1. **统一位置名称**：LW、RW、LM、RM、Winger 归为 `Winger`；CM、AM、DM 保留细分位置。
2. **年龄筛选**：只保留 `age <= 21` 的球员；如果只有 `date_of_birth`，则根据比赛赛季或当前日期计算年龄。
3. **出场时间筛选**：正式评分默认要求 `minutes >= 600`。低于 600 分钟的球员可以进入**观察名单**，但不进入 Top 20 正式榜。
4. **缺失值处理**：
   - 必须字段缺失 → 剔除或标记为不可评分。
   - 非关键计数字段缺失 → 可填 0。
   - 百分比字段缺失 → 保持为空并降低数据完整度。
5. **去重**：优先按 `player_name + team + league` 去重；如果有 `player_id`，则优先使用 `player_id`。
6. **统一数值格式**：百分比统一转为 0-100 或 0-1 中的一种，并在代码中保持一致。
7. **计算每 90 分钟数据**：如 `goals_per90 = goals / minutes * 90`（详细放在阶段 3）。

## 位置映射（position_mapper.py）

> **方案 B 重要变化**：数据来自 ScraperFC 采集的 FBref，其赛季表位置只有粗分类 `FW / MF / DF / GK`，**无法直接得到 Winger / AM / CM / DM**。因此细分位置在**阶段 0** 已用 **Transfermarkt 主位置**补齐（见 [阶段0](./阶段0-数据采集与位置补充.md)）。本阶段的 `position_mapper.py` 优先使用该细分位置，FBref 粗位置仅作兜底。

**位置来源优先级**：
1. Transfermarkt 主位置（阶段 0 补齐，命名近 1:1）→ 经 `position_mapping.yaml` 映射
2. FBref 个人页二级位置（兜底 B，仅 Top 20 候选）
3. FBref 粗位置 + 启发式（第 0 层兜底，匹配不上时使用）

| 输入位置名（Transfermarkt 主位置） | 标准位置 standard_position |
|-----------|---------------------------|
| Right Winger, Left Winger, Winger | Winger |
| Central Midfield | CM |
| Attacking Midfield | AM |
| Defensive Midfield | DM |

映射配置 `config/position_mapping.yaml`：

```yaml
Right Winger: Winger
Left Winger: Winger
Winger: Winger
Attacking Midfield: AM
Central Midfield: CM
Defensive Midfield: DM
```

位置分组配置（来自 `config.yaml`，用于筛选进入评分的位置）：

```yaml
positions:
  winger: [Winger]
  midfield: [CM, AM, DM]
```

## 实现要点

- `cleaner.py` 负责年龄/分钟数筛选、缺失值处理、去重、数值格式统一。
- `position_mapper.py` 优先取 Transfermarkt 主位置映射为 `standard_position`（Winger/CM/AM/DM），未匹配上时回落 FBref 粗位置 + 启发式，并在数据完整度里标记。
- 清洗后写出 `cleaned_players.csv`，供后续阶段复用。
- 保留「观察名单」与「正式评分」的区分：分钟数 < 600 不进正式榜，但不必删除。

## 相关风险

| 风险 | 表现 | 解决方案 |
|------|------|----------|
| 小样本虚高 | 出场很少但 per90 数据很夸张 | 正式评分要求 `minutes >= 600` |

## 验收标准

- [ ] 输出数据中不存在 `age > 21` 的正式评分球员。
- [ ] 正式评分数据只包含 Winger、CM、AM、DM。
- [ ] （配合阶段 6）Top 20 中不存在 `minutes < 600` 的球员。

## 学习路径提示

- 学会筛选数据：`age <= 21`、`minutes >= 600`、`position` 是否在指定列表中。
