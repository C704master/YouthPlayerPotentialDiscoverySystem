# 开发文档索引

> 本目录由《年轻球员潜力发现与报告生成系统 · 技术方案设计文档（方案 A：本地 Python 脚本版 MVP）》拆分而来，按开发阶段组织，便于逐步落地实现。

## 核心结论

第一版不做网站、不训练深度学习模型；先完成
**「数据采集 → 数据清洗 → 潜力评分 → Top 20 排行榜 → 雷达图 → Markdown/Word 报告包」** 的完整闭环。

> **数据来源（方案 B）**：采用 [ScraperFC](https://github.com/oseymour/ScraperFC) 采集 FBref（10 联赛 × 3 赛季）+ Transfermarkt 主位置，作为**阶段 0** 产出 `data/raw/players.csv`。详见 [阶段0](./阶段0-数据采集与位置补充.md) 与 [`F&Q/方案B-数据采集实现思路.md`](../F&Q/方案B-数据采集实现思路.md)。

## 阅读建议

1. 先读 [00-总览与架构](./00-总览与架构.md) 了解系统边界、总体数据流和目录结构。
2. 按阶段 1→9 顺序实现，每个阶段文档内含：阶段目标、输入/输出、模块职责、详细规则、验收标准、相关风险。
3. 每完成一个阶段就核对该阶段的「验收标准」，通过后再进入下一阶段。
