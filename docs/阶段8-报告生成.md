# 阶段 8 · 报告生成

> 对应原文档：五、核心模块设计（prompt_builder.py、report_generator.py、word_exporter.py）／ 十、报告生成方案（10.1 大模型使用规则）／ 十六、风险与解决方案（大模型幻觉）

## 阶段目标

为 Top 20 每名球员生成 **Markdown 源报告** 与 **Word 报告**。

报告生成分两层：**先生成 Markdown 源稿，再导出 Word**。Markdown 方便检查和调试，Word 适合最终提交和答辩展示。

## 输入 / 输出

| 项 | 内容 |
|----|------|
| 输入 | 结构化球员数据、分项评分、雷达图 PNG（阶段 7）、`templates/player_report.md`、`templates/llm_prompt.txt` |
| 输出 | `outputs/reports_md/player_name.md`、`outputs/reports_docx/player_name.docx` |
| 模块 | `prompt_builder.py`、`report_generator.py`、`word_exporter.py` |

## 报告章节结构

| 报告章节 | 内容要求 |
|----------|----------|
| 1. 球员基本信息 | 姓名、年龄、位置、球队、联赛、国籍、身价等可用字段 |
| 2. 当前表现概况 | 出场时间、主要数据、总评分、分项评分 |
| 3. 潜力评分解释 | 说明为什么得分高或低，必须基于数据 |
| 4. 主要优势 | 用 goals_per90、key_passes_per90、dribbles_per90 等数据解释 |
| 5. 主要短板 | 用数据说明不足，不编造主观判断 |
| 6. 场上行为风格画像 | 只分析场上行为倾向，不做心理诊断 |
| 7. 发展风险 | 样本量、纪律、稳定性、数据完整度、位置适配等 |
| 8. 适合的球队或战术类型 | 基于数据谨慎判断 |
| 9. 一句话总结 | 简洁概括球员类型 |
| 10. 数据限制说明 | 明确说明缺失字段和结论谨慎性 |

## 10.1 大模型使用规则（强约束）

- 大模型只负责**解释数据和生成自然语言报告**，不直接决定潜力评分。
- 报告必须基于程序提供的数据和分项评分。
- **不得编造**伤病、转会、采访、家庭背景、心理状态或真实性格。
- 如果数据不足，必须写明「数据不足，结论需谨慎」。
- 必须区分事实和推测，例如使用「数据显示」「可能说明」等表达。

大模型提示词核心约束示例（`templates/llm_prompt.txt`）：

```
你是一名足球数据分析助理。请只根据我提供的球员数据、评分结果和雷达图说明生成报告。
不要编造任何未提供的信息，包括伤病、转会传闻、采访内容、家庭背景或心理性格判断。
如果字段缺失，请明确说明数据不足。
「场上行为风格画像」只能基于比赛数据分析场上倾向，不构成心理诊断。
```

## 实现要点

- `prompt_builder.py`：把球员结构化数据 + 分项评分组织成受约束的提示词。
- `report_generator.py`：用 Jinja2 或字符串模板生成 Markdown 源稿（可先不接大模型，用模板 + 数据填充；后续再接入 LLM 生成自然语言段落）。
- `word_exporter.py`：用 `python-docx` 把内容 + 雷达图导出为 `.docx`。
- 大模型可用 OpenAI API 或**手动粘贴提示词**两种方式，第一版均可。

## 相关风险

| 风险 | 表现 | 解决方案 |
|------|------|----------|
| 大模型幻觉 | 生成未提供的信息 | 提示词强约束 + 报告中必须写数据限制 |

## 验收标准

- [ ] Top 20 每名球员生成单独的 DOCX 文件。
- [ ] 报告不出现未提供的伤病、转会、采访或心理判断。
- [ ] 数据不足时报告明确写出「数据不足，结论需谨慎」。

## 学习路径提示

- 学会 `python-docx` 生成 Word；最后再接入大模型生成自然语言报告。
