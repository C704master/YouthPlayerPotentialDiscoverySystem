# 阶段 3 · 特征计算

> 对应原文档：五、核心模块设计（feature_builder.py）／ 七、数据清洗规则（每 90 分钟数据）／ 六、数据字段设计（数据完整度）

## 阶段目标

能计算每 90 分钟（per90）效率数据和**数据完整度**，为后续同位置百分位评分做准备。

## 输入 / 输出

| 项 | 内容 |
|----|------|
| 输入 | 阶段 2 的 `cleaned_players.csv` |
| 输出 | 增强字段后的 DataFrame（含各类 `*_per90` 与 `data_completeness`） |
| 模块 | `feature_builder.py` |

## per90 指标计算

把累计数据换算成每 90 分钟效率，消除出场时间差异带来的不可比：

```
goals_per90       = goals / minutes * 90
assists_per90     = assists / minutes * 90
key_passes_per90  = key_passes / minutes * 90
dribbles_per90    = dribbles / minutes * 90
...（对所有核心计数字段同理）
```

建议为以下核心字段生成 per90 版本（视数据可得性）：
goals、assists、shots、key_passes、dribbles、passes、tackles、interceptions，
以及增强字段 progressive_carries、shot_creating_actions 等。

## 数据完整度（data_completeness）

用于在评分和报告中提示「这名球员的数据有多完整」。

- 核心评分字段缺失越多 → 完整度越低。
- 百分比字段缺失 → 保持为空并降低完整度。
- 完整度低会在阶段 4 参与风险扣分，并在报告中提示结论需谨慎。

建议实现：统计每名球员在「核心评分字段」清单中实际有值的比例，得到 0-1 或 0-100 的完整度分。

## 实现要点

- `feature_builder.py` 接收清洗后的数据，逐列生成 per90 派生列。
- 分母 `minutes` 为 0 或缺失时要保护，避免除零错误。
- 同时计算 `data_completeness`，作为阶段 4 风险扣分与报告提示的依据。
- 输出增强后的 DataFrame，供阶段 4 评分使用。

## 相关风险

| 风险 | 表现 | 解决方案 |
|------|------|----------|
| 数据缺失 | 某些高级字段没有，如 xG、progressive_carries | 字段分级 + 数据完整度，缺失时降低完整度而非阻塞 |

## 验收标准

- [ ] （配合阶段 4）评分结果中体现基于 per90 的核心表现。
- [ ] 每名球员具备 `data_completeness` 指标，供风险扣分和报告使用。
